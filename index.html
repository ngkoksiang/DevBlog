Week 01: HTML

  • "P" is a paragraph tag
  • "table" tag creates a table with "tr" for rows and "tc" for columns
  • "form" tag for creating forms with different types such as name, email, date, etc.
  • "ol" for ordered lists and "ul" for unordered lists
  • "b" for bold text, "br" for line breaks
  • "div" for creating divisions, "h1-h6" for headers, and "footer" for last division on body
  • "nav" for navigation bar and "img" for images

Week 02: CSS

  • Link CSS to HTML using "link rel='stylesheet' href='{directory of CSS file}'"
  • Change web page content, color, text alignment, border, background, width/height, font, margin/padding
  • Use period (.) for class and hash (#) for ID to call attributes in CSS

Week 03: HTML & CSS

  • "input" tag for creating form with different types such as text, password, date, checkbox, etc.
  • "button" tag for clickable buttons, "required" for form validation, "placeholder" for form text
  • "textarea" for text boxes, positioning elements with relative, absolute, fixed positioning
  • Flexbox and "flex" for layout design

Week 04: Github & Responsive Images

  • Setting up Github, making images responsive with "max" and "min" CSS attributes

Week 05: JavaScript

  • 8-hour course on Brightspace covers a lot of content

Week 06: JavaScript

  • JavaScript objects with properties and methods
  • "if" and "else" statements, "var" for variables, "console.log" to view results in console
  • Change properties with "varname.property", "for" loops

Week 07: DOM & JavaScript

  • Document Object Model (DOM) allows access to HTML elements
  • Use "getElementById", "querySelector", "getElementsByClassName"

Week 08: jQuery & API

  • Use "querySelector" and "querySelectorAll" to select specific elements
  • Edit CSS styles, add HTML tags, and shorten JavaScript codes

Week 09: Bootstrap

Bootstrap is a library of pre-made CSS to make coding easier, reducing the need for writing custom CSS.

Week 12: API access and automation

APIs provide access to data from databases, cloud services, and other sources. They can also be used to automate processes and perform tasks without human intervention. The ability to post and delete data is also important in API usage.

Week 13: Best Practices for Website Development

  • Ensure meaningful GitHub commit messages
  • Properly format README.md
  • Validate HTML and CSS
  • Properly structure files in folders with meaningful naming conventions
